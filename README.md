# Sitio de estudio para administracion de sistemas

Este repositorio ya esta organizado para subirse directamente a GitHub como un sitio estatico.

## Estructura

- `index.html`: pagina principal del sitio.
- `material/resumen-estudio.txt`: resumen del contenido.
- `material/transcripcion-material.txt`: transcripcion completa organizada.
- `material/comandos-linux-practica.docx`: material adicional de comandos Linux.
- `material/semana-5-automatizacion-linux.docx`: material adicional de automatizacion en GNU/Linux.
- `material/material-original.pdf`: PDF original.
- `material/imagenes/`: imagenes fuente renombradas por pagina.

## Subir a GitHub

1. Crea un repositorio nuevo en GitHub.
2. Sube el contenido completo de esta carpeta.
3. Asegurate de que `index.html` quede en la raiz del repositorio.

## Publicar con GitHub Pages

1. En GitHub entra a `Settings` del repositorio.
2. Abre la seccion `Pages`.
3. En `Build and deployment`, elige `Deploy from a branch`.
4. Selecciona la rama principal y la carpeta `/ (root)`.
5. Guarda los cambios.

Cuando GitHub termine la publicacion, el sitio quedara listo usando `index.html` como pagina principal.
